# Baudoku — Design Context

> One-time design context for the Baudoku Expo app. Loaded by `/impeccable` and `shape` skills before any UI work.

## Users

**Primary persona: Sachverständiger (independent appraiser / building inspector).**

- Visits construction sites to document defects ("Mängel") for formal expert reports ("Gutachten").
- Works on a 12.9"-class tablet on site (primary) and reviews/exports on the same device or phone later.
- Often offline (basement, new build with no signal). Sync happens later.
- Output is a credible report sent to clients, lawyers, courts, or insurers. Accuracy and traceability matter more than speed.

**Job to be done:** capture every defect (text + voice + photo + plan marker) without losing a single observation, then assemble a defensible report.

## Brand Personality

**Three words: precise · durable · calm.**

- **Precise** — reads like a measuring instrument. No decorative noise. Numbers, statuses, and severities have one obvious meaning. Type and spacing feel engineered, not artful.
- **Durable** — solid surfaces, restrained palette, comfortable weight. Looks like a tool that survives a construction site, not a consumer app. No fragile thin strokes.
- **Calm** — no urgency tics, no celebratory animation. Methodical pacing. The interface waits for the user; it doesn't push.

## Aesthetic Direction

**Theme:** light primary (glare-readable on outdoor tablets in daylight), dark optional. Both implementations must hit AA contrast minimum, prefer AAA for body text on the primary surface.

**Visual reach (positive):**
- DIN-adjacent technical typography, mechanical engineering drawing aesthetic, German industrial signage, observation log books, architectural stamp/seal craft, durable utility objects (Sennheiser, Leica, Festool, Wenger).
- Surfaces feel like measured materials: matte paper, anodized aluminum, calibrated grids.

**Hard avoids:**
- Generic SaaS dashboard aesthetic (rounded card grid + soft drop shadow + Inter).
- Consumer fitness/social vibe (vibrant gradients, big celebratory CTAs).
- Construction-stereotype hi-vis orange / safety-vest yellow as primary.
- Crypto / AI gradient (cyan→purple, neon-on-dark, glassmorphism).
- Government-portal blandness (pure-gray utility, accessibility checkbox aesthetic).

## Design Principles

1. **Information clarity over decoration.** Every glyph, divider, and color earns its place by aiding observation, severity reading, or report integrity. Decoration that does not aid the inspection workflow is removed.
2. **Glare-readable defaults.** Light theme is primary because it survives outdoor daylight on a tablet. High contrast for text. No body type below 15px. No thin weights for ambient text.
3. **Methodical pacing, not eager.** Motion is functional: state transitions, sync feedback, gesture affordances. No bounce, no celebratory flourishes, no decorative loops. Standard easing only. ≤220ms for state changes; ≤120ms for input feedback.
4. **Durable visual language.** Solid surfaces over glass. Full borders or no borders — no left-stripe accent. Tinted neutrals (warm-cool toward the brand hue), no pure black/white. Generous spacing rhythm; nothing squashed.
5. **Trade-precise German microcopy in du-form.** Building-trade vocabulary preferred over generic UX phrasing. Direct, observation-style ("Mangel erfassen", "Foto ergänzen") not marketing ("Lass uns starten!"). Errors describe what happened and what to try, not "Oops!".
6. **Severity is the only loud color.** Defect priority and status warrant saturation. Everything else is calibrated neutrals + one restrained brand accent. Color is used as data, not decoration.
7. **Offline is first-class.** Sync state must be visible at a glance, not buried. The user trusts the app with field-only observations; that trust is earned by always showing where data lives.
8. **Tablet-first, phone-graceful.** Layouts are designed for landscape 12.9" tablet first (master-detail, persistent rails). Phone is a graceful collapse, not a separate design.

## Locked Constraints

- Stack: Expo 54, React Native 0.81, React 19, Reanimated 4, Supabase, expo-sqlite (offline-first).
- Form validation: zod (approved).
- Storybook-style dev preview screen: yes, gated by `__DEV__`.
- Microcopy: du-form, German.
- Primary device: 12.9" tablet (landscape + portrait). Phone supported.
- A11y baseline: WCAG AA minimum, AAA preferred for body text. accessibilityLabel + maxFontSizeMultiplier required on every interactive primitive.
- Anti-bans (from impeccable absolute_bans): no left-stripe borders >1px, no gradient text, no glassmorphism, no Inter / Plex / DM / Outfit / Fraunces / Newsreader / Crimson / Cormorant / Space Grotesk / Instrument default fonts. Display + body pair must be selected from a fresh search per Step 3 of font_selection_procedure.

## Open Decisions Deferred to Phase 1 Design Brief

- Final type pair (display + body) — to be selected during shape interview against the three brand words.
- Brand accent hue — to be derived from "precise · durable · calm" plus light-theme glare requirement (likely a deep, low-chroma blue-graphite or warm steel; not navy SaaS blue).
- Severity color ramp (info / low / medium / high / critical) — must be colorblind-distinguishable and printable in the exported PDF report.
- Master-detail breakpoint pixel value — current proposal `width >= 900` from audit; verify against real iPad mini / 11" / 12.9" widths.
